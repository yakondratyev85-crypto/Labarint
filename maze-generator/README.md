# Генератор лабиринтов SVG

Это полностью локальное веб-приложение для генерации лабиринтов разных типов и сложности с экспортом в чистый векторный SVG.

Проект не использует npm, Vite, React, Vue, Angular, TypeScript, Webpack, серверы или внешние библиотеки. Всё работает в обычном браузере из статических файлов.

## Как запустить

1. Скачайте репозиторий, например через GitHub Desktop.
2. Откройте папку `maze-generator`.
3. Дважды кликните по файлу `index.html`.
4. Генератор откроется в браузере и сразу построит первый лабиринт.

Сервер запускать не нужно.

## Возможности

- Генерация лабиринтов по выбранным настройкам.
- Предпросмотр результата прямо на странице.
- Повторяемая генерация через seed.
- Кнопка `Random Seed` для быстрого случайного seed.
- 5 уровней сложности: `Very Easy`, `Easy`, `Medium`, `Hard`, `Expert`.
- Режимы генерации: `Classic`, `Kids Easy`, `Puzzle`, `Long Path`, `Print Clean`.
- Возрастные пресеты: `4–5 years`, `6–7 years`, `8–9 years`, `10+ years`, `Adults`.
- Настройка положения старта и финиша.
- Настройка ширины, высоты, толщины стен и размера клетки.
- Поиск решения через BFS.
- Показ и скрытие решения.
- Экспорт SVG без решения.
- Экспорт SVG с решением.
- Информационный блок с параметрами, длиной решения, количеством поворотов, оценкой сложности и размером SVG.

## Поддерживаемые типы лабиринтов

1. **Rectangular Maze** — классический прямоугольный лабиринт на квадратной сетке. Генерируется алгоритмом DFS / Recursive Backtracking.
2. **Circular Maze** — круговой лабиринт из концентрических колец и радиальных перегородок. Стены строятся SVG-элементами `path`, `circle` и `line`.
3. **Hex Maze** — лабиринт на шестиугольной сетке, где у каждой клетки до 6 направлений.
4. **Room / Island Maze** — прямоугольный лабиринт с дополнительными комнатами, открытыми зонами, островами и проходами. Путь от старта до финиша сохраняется всегда.
5. **Shape Maze** — лабиринт внутри формы. Поддерживаются формы `circle`, `heart` и `star`; клетки вне формы не отображаются.

## Этап 1 — усиление основы

На этапе 1 проект был усилен без добавления сборки, сервера или зависимостей.

### Maze Mode

Режимы реально влияют на генерацию:

- **Classic** — обычный честный лабиринт с одним стартом, одним финишем и гарантированным путём.
- **Kids Easy** — больше дополнительных соединений, меньше тупиков, крупнее и читабельнее проходы; подходит для детей 4–6 лет.
- **Puzzle** — меньше лишних соединений, больше тупиков и развилок, более длинный и сложный маршрут.
- **Long Path** — старт и финиш подбираются по дальним точкам графа, чтобы путь был длиннее.
- **Print Clean** — умеренная плотность, аккуратные отступы, чёткие линии и чистая SVG-структура для печати.

### Улучшенная сложность

Сложность задаётся объектом `difficultyPresets`. Каждый уровень содержит параметры:

- `gridSize` — рекомендуемый размер сетки;
- `deadEndFactor` — ожидаемая доля тупиков;
- `branchFactor` — интенсивность развилок;
- `extraConnections` — дополнительные проходы;
- `pathComplexity` — сложность основного маршрута;
- `minSolutionLength` — минимальная желаемая длина решения;
- `corridorWidthStyle` — стиль читаемости проходов.

`Very Easy` создаёт маленькие и простые лабиринты, а `Expert` — более крупные и плотные.

### Возрастные пресеты

Выбор `Age Preset` автоматически выставляет рекомендуемые значения сложности, режима, размера, толщины стен, размера клетки и позиции старта/финиша:

- **4–5 years** — простая сетка, мало тупиков, толстые линии, крупные клетки.
- **6–7 years** — чуть больше клеток, но режим остаётся дружелюбным.
- **8–9 years** — средняя сложность и понятная структура.
- **10+ years** — более сложные маршруты и больше развилок.
- **Adults** — высокая сложность, больше клеток и тоньше линии.

После выбора пресета пользователь может вручную изменить любые настройки.

### Старт и финиш

Настройка `Start / Finish Position` поддерживает варианты:

- `corners` — старт около верхнего левого угла, финиш около нижнего правого;
- `left-to-right` — старт на левой границе, финиш на правой;
- `top-to-bottom` — старт сверху, финиш снизу;
- `random edges` — случайные разные границы;
- `longest path` — две удалённые точки графа лабиринта.

### Решение и статистика

Решение строится как один SVG `path`, проходит по центрам клеток и находится через BFS. Функция `getSolutionStats(solution)` возвращает:

- `solutionLength`;
- `turnsCount`;
- `deadEndsCount`;
- `estimatedDifficultyScore`.

### Validation

После генерации вызывается `validateMaze(maze)`. Проверяется:

- наличие старта и финиша;
- старт и финиш не совпадают;
- существует путь от старта до финиша;
- решение не пустое;
- нет полностью изолированных активных клеток;
- SVG стен не пустой;
- количество стен больше нуля.

Если генерация не прошла проверку, приложение делает до 10 повторных попыток. Если и они не помогли, в интерфейсе показывается понятная ошибка, а приложение не падает.

### Чистый SVG-экспорт

Экспортируемый SVG остаётся чистым вектором и содержит слои:

```xml
<g id="maze-background">...</g>
<g id="maze-frame">...</g>
<g id="maze-walls">...</g>
<g id="maze-start-finish">...</g>
<g id="maze-solution">...</g>
<g id="maze-labels">...</g>
```

В SVG нет PNG, JPG, base64, canvas bitmap, `foreignObject` или растровых вставок.

## Как скачать SVG

1. Настройте тип лабиринта, режим, возрастной пресет, сложность, размер, толщину стен, размер клетки и seed.
2. Нажмите `Generate`.
3. Для файла без решения нажмите `Download SVG`.
4. Для файла с красной линией решения нажмите `Download SVG with Solution`.

SVG без решения не содержит `solution-path`. SVG с решением содержит решение в отдельном слое `maze-solution`.

Файлы можно открывать в браузере, Figma, Illustrator и Inkscape.

## Seed и повторяемость

При одинаковом seed и одинаковых настройках генератор создаёт одинаковый лабиринт. Если изменить seed, структура лабиринта изменится. Для детерминированной генерации используется seeded random на основе `mulberry32`.
